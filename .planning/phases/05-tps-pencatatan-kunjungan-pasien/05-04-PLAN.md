<task type="auto">
  <name>Task 1: Implement createResep handler with database transaction</name>
  <files>backend/src/controllers/tps/resep.ts</files>
  <read_first>backend/src/models/Resep.ts</read_first>
  <action>
    Create backend/src/controllers/tps/resep.ts.
    Implement `createResep`:
      - Validate that the user is an `apoteker` or `admin`. Return 403 if not.
      - Retrieve the `RekamMedis` visit by ID and check that it belongs to the user's `faskes_id`.
      - Check if the visit already has a prescription. If so, return a 409 status code.
      - Validate the prescription items:
        - Check XOR for `obat_id` and `formula_id` (only one must be present per item).
        - Verify stock availability in the user's faskes for each item (for `obat_id` direct check; for `formula_id`, query components and check aggregate quantities).
        - If any item has insufficient stock, return a 400 error with the list of insufficient items.
      - Execute the transaction:
        - Save `Resep` header (`dibuat_oleh = req.user.id`).
        - For each item, save `ResepItem`.
        - Deduct stock in the `Stok` table for each medicine or formula component.
        - Create audit trail entries in `PergerakanStok` with type `keluar`, source `faskes_asal = faskes_id`, destination `faskes_tujuan = null`, referencing `resep.id`.
      - Return 201 with prescription details.
  </action>
  <verify>cd backend && npx tsc --noEmit</verify>
  <acceptance_criteria>
    - resep.ts compiles cleanly.
    - Otorisasi apoteker/admin is enforced.
    - Transaction commits or rollbacks correctly depending on stock availability.
  </acceptance_criteria>
  <done>createResep controller implemented.</done>
</task>
