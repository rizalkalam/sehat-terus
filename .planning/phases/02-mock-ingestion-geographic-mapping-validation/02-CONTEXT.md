# Phase 2: Mock Ingestion & Geographic Mapping Validation - Context

**Gathered:** 2026-06-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Integrating a valid, simplified Sleman Regency (Daerah Istimewa Yogyakarta) kecamatan GeoJSON boundary file, ensuring it contains sub-districts like Mlati, Ngemplak, Depok, and Gamping. Implementing a CLI seeder script using Sequelize and Faker.js in the backend to generate and insert at least 5,000 realistic clinical records where sub-district names map exactly to the keys in the Sleman GeoJSON.

</domain>

<decisions>
## Implementation Decisions

### GeoJSON Region
- **D-2.1:** Use **Sleman Regency (Daerah Istimewa Yogyakarta)** kecamatan boundaries as the geographic model. This contains the 17 sub-districts of Sleman (Depok, Mlati, Ngemplak, Gamping, etc.) which matches the kecamatan already referenced in the frontend prototype.

### GeoJSON File Location
- **D-2.2:** Save the Sleman kecamatan GeoJSON file under **`frontend/public/geojson/sleman-kecamatan.geojson`** so the Next.js server can serve it statically to the react-leaflet map.

### Seeder Script
- **D-2.3:** Create a Sequelize-based seeder script in the Express.js backend at **`backend/src/seeders/20260622000000-rekam-medis-seeder.ts`** or as a standalone TS script. We will run it via a command in `package.json` (e.g., `npm run seed`).

### Sub-district Name Alignment
- **D-2.4:** The seeder script must pull sub-district names from a fixed array of the 17 Sleman sub-districts matching the exact names (case-sensitive) in the GeoJSON `WADMKC` or `kecamatan` properties.

</decisions>

<specifics>
## Specific Ideas

- The seeder will generate clinical records covering the last 12 months using valid ICD-10 codes for common diseases like Influenza (J11), DBD (A90), Ispa (J06.9), Diare (A09), and Darah Tinggi (I10).
- A Node.js validator script will check that all sub-district names generated by the seeder are present in the GeoJSON file.

</specifics>
